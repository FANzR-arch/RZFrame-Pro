# 指标体系（北极星 → 漏斗）
建议以“交付成功效率”作为北极星：用户是否能在短时间内完成高质量导出。该目标在现有文档里可拆成三类指标：  
1. **激活指标**：TTFV（首次价值时间）与首次导出成功率。  
2. **留存指标**：周使用频次、D1/D7 回访率、二次打开率。  
3. **商业指标**：付费点击率、付费转化率、功能解锁使用率。  
漏斗建议统一为：曝光 → 访问/下载 → 安装启动 → 首次导出 → 周内复用 → 付费转化。当前仓库内缺少统一的真实数值口径，除版本记录外，多数指标数据为【待补充】。

## 验证记录（假设→实验→结果→结论）
1. **假设**：自动品牌识别能显著减少手工设置步骤。  
**实验**：在 `v2.7.3` 上线自动品牌识别 + Logo 循环切换。  
**结果**：功能已稳定上线并进入主流程；量化“步骤减少幅度”暂无统一数据。  
**结论**：方向成立，需补埋点验证真实效率收益。  

2. **假设**：模板资产化能提高复用率与一致性。  
**实验**：提供模板保存、导入导出能力（`.rzf`/JSON 体系）。  
**结果**：功能可用，用户复用率与模板传播数据【待补充】。  
**结论**：可作为留存抓手，但需补量化证据。  

3. **假设**：首用入口优化会提升激活率。  
**实验**：审计中提出“下载与使用”优先、默认模板可直接出图。  
**结果**：文档侧已改进，应用内首用行为数据【待补充】。  
**结论**：需结合安装-首次导出漏斗继续验证。  

4. **假设**：付费应围绕效率与品牌能力，而非画质阉割。  
**实验**：增长方案提出“Pro Features”点击意愿测试（CTR）路径。  
**结果**：当前未见可核验实验数据。  
**结论**：作为下一阶段重点验证项推进。

## 风险与约束（合规/成本/体验/增长）
1. **竞争风险**：若大厂在核心工作流内原生覆盖同类能力，工具型差异会被快速抹平。  
2. **增长风险**：桌面端传播链路较长，天然弱于移动端“拍-修-发”一体链路。  
3. **数据风险**：Exif 长尾兼容与老镜头异常会影响专业可信度，需持续维护映射与纠错机制。  
4. **体验风险**：高分图批处理存在性能与稳定性压力，需持续进行内存与渲染优化。  
5. **隐私与合规风险**：元数据可能包含位置信息，需明确默认策略与用户可见开关。  
6. **经营风险**：当前缺少完整的量化看板，策略优先级判断仍有不确定性。
