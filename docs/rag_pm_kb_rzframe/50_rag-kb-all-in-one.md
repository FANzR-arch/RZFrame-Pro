# 这是什么（结论）
RZFrame 是一个面向摄影师的桌面端“作品交付最后一公里”工具：在不上传云端的前提下，把照片自动封装成带 EXIF 参数与品牌表达的可发布成品。它不是单纯“加边框特效”，而是把“修图完成后到对外发布前”的重复劳动标准化，让用户用更少时间输出更稳定的专业呈现。当前版本已到 `v2.7.3`，核心闭环是“拖入照片 → 自动识别参数与品牌 → 选择模板微调 → 单图/批量导出”。

## 为什么做（问题与证据）
问题不是“能不能加框”，而是“为什么最后一步总是又慢又碎”。项目文档显示，摄影师前期拍摄和修图投入大，但分享时常回到手工拼装：在 Photoshop 手动排参数，单张约 5-10 分钟；移动端工具虽然便捷，但在高分图、批处理和本地隐私上难满足专业用户。另一个高频冲突是平台比例规则与原始构图冲突，用户需要一个可控“安全区”来避免内容被平台粗暴裁切。RZFrame 的切入点就是把这些痛点从“手工操作”改成“可复用流程”。

## 目标与指标（北极星/阶段指标）
北极星目标是“让用户在一次编辑中稳定完成高质量交付”，对应指标可定义为首次导出成功率与 TTFV（首次价值时间）。阶段性指标在现有文档中已明确：`TTFV < 30 秒`、自动品牌匹配后无需手动切换占比 `> 85%`、D7 留存相对基线提升 `15%`（规划目标）。目前可公开的真实漏斗数据（曝光、下载、激活、留存、付费）尚未在仓库内形成可核验口径，统一标注为【待补充】。

## 我做了哪些关键决策（3–6条，含取舍理由）
1. **本地优先，不走云端处理**：优先满足专业用户对隐私与原图安全的硬约束；取舍是放弃云协同带来的轻分享能力。  
2. **先做“可交付闭环”，再做“创意扩展”**：P0 先保证导入、解析、预览、导出全链路可用；取舍是延后账号、社区、社媒直发等高复杂功能。  
3. **自动化优先于手工配置**：做自动品牌识别、Logo 循环与持久化，减少首用学习成本；取舍是前期需要更高的识别与兼容维护成本。  
4. **模板化而非自由拼装**：先用 Classic/Cinema/Float 三套模板覆盖主场景，保证稳定体验；取舍是暂不追求无限视觉自由度。  
5. **技术只服务产品目标**：采用 Electron + ExifTool + Canvas，核心理由是兼顾本地 I/O、参数解析可靠性与高分图渲染性能，不追求技术炫技。

## 方案概览（用户主流程 + 核心功能）
主流程控制在 5 步内：导入照片、自动读取 Exif、自动匹配品牌样式、模板与参数微调、导出成品。核心能力围绕“效率与一致性”展开：批量导入与批量导出、参数应用到全部、模板保存与导入导出、Logo 的自动匹配与手动兜底、跨语言界面支持。对用户价值而言，关键不是多一个按钮，而是把“重复动作”沉淀为“默认动作”，让每次发布都可复用、可预测、可控。

## 验证与结果（证据/数据/反馈）
已验证结果以“版本落地 + 功能可用”为主：`v2.7.2 -> v2.7.3` 已上线自动品牌识别、Logo 循环、Logo 持久化、Logo 文件夹导入，并在审计文档中确认核心功能稳定。日志文件可看到真实导入、解析、交互、导出链路记录，说明主流程已跑通。尚缺的是产品指标层面的闭环数据（如首次导出成功率、导出时长分布、D1/D7 留存、付费转化），因此所有“增长效果”目前只能视为假设或目标，需补齐埋点后再做结论。

## 复盘（做对了什么/做错了什么/学到了什么）
做对的部分是抓住了明确人群与明确场景：不是做大众滤镜，而是服务“重画质、重效率、重表达一致性”的摄影用户；同时版本迭代方向从“功能堆叠”转向“流程缩短”。做得不够的是数据资产建设滞后：很多策略有清晰假设，但缺少统一口径指标来证明“真的更好”。最大的学习是：工具产品要从“功能可用”走向“工作流可复用”，必须把体验路径和数据路径同时建设，否则难形成可持续的产品叙事与商业叙事。

## 下一步（最值得做的 3 件事）
1. **补齐指标与埋点基线**：统一定义曝光、下载、激活、导出成功率、D1/D7 留存、付费转化口径，先完成可观测。  
2. **优化首用 30 秒路径**：围绕“拖入即出图”继续减少认知与操作步骤，优先验证首次导出成功率。  
3. **从生成器升级为工作流节点**：推进历史库/资产管理/流程化入口（如监听目录等）能力，提升留存与付费意愿。

# 用户是谁？核心场景是什么？
RZFrame 当前服务的是“有明确交付需求的摄影用户”，而不是泛娱乐修图人群。第一类是职业摄影师，典型场景是修图完成后要快速输出一批可交付样片；第二类是摄影博主或 KOL，典型场景是在社交媒体发图时既要保留构图，也要表达拍摄参数与器材身份；第三类是胶片与器材爱好者，核心诉求是长期归档时保留参数语义和风格一致性；第四类是器材评测者，重点在于标准化展示不同机身、镜头组合。共同点是：他们已投入高成本完成“拍摄与修图”，最怕最后一步变慢、变乱、变不专业。

## 痛点清单（带证据）
1. **分享链路碎片化**：拍摄与修图后还要手工拼参数，流程断裂。证据：`docs/SHOWCASE_RZFRAME.md` 提到 Photoshop 手工加框单张约 5-10 分钟。  
2. **画质与便捷冲突**：移动端工具便捷，但专业用户担心高分图压缩、色彩损失。证据：`docs/Product_Introduction.md` 与项目自检文档强调 50MP+ 场景与无损诉求。  
3. **平台比例与构图冲突**：社媒比例规则导致原图构图被裁切。证据：`docs/项目自检/摄影边框的专业价值与市场逻辑.md` 明确“边框作为安全区”价值。  
4. **首用门槛风险**：早期文档偏开发者视角，非技术用户可能卡在入口。证据：`docs/AUDIT_DEEP_20260202.md` 对“神奇前30秒”提出风险。  
5. **专业感与身份表达不足**：用户不仅要“发图”，还要可见的专业表达与一致品牌风格。证据：`docs/SHOWCASE_RZFRAME.md` 与增长文档多次强调“品牌化封装”。

## 需求列表（用“用户故事/Job to be done”写）
1. 作为摄影师，我希望把多张照片一次性导入并批量导出，以便减少重复劳动。  
2. 作为摄影师，我希望系统自动读取 Exif 参数并展示关键字段，以便不用手工抄写。  
3. 作为器材用户，我希望系统自动识别品牌并匹配 Logo，以便快速形成专业展示。  
4. 作为创作者，我希望可以在模板内调节字体、边框、颜色、比例，以便统一视觉风格。  
5. 作为运营型创作者，我希望把当前设置应用到全部图片，以便提高出图一致性。  
6. 作为长期用户，我希望保存并导入导出模板，以便跨项目复用风格资产。  
7. 作为隐私敏感用户，我希望所有处理在本地完成，以便避免上传原图风险。  
8. 作为多语言用户，我希望可切换 CN/EN/JP，以便降低学习成本。  
9. 作为进阶用户，我希望异常情况下有兜底方式（如 Logo 缺失时文本回退），以便不中断交付。  
10. 作为产品经理，我希望拿到可追踪的指标闭环，以便判断功能优先级是否正确【待补充】。

## 优先级与取舍（RICE/影响力-成本/风险矩阵均可，用文字解释即可）
当前优先级策略可归纳为“影响力优先 + 成本可控 + 风险可兜底”。  
P0 聚焦导入、解析、模板预览、导出、批处理、自动 Logo 等核心闭环，因为这些能力直接决定用户是否能完成任务。  
P1 放在模板资产化、多语言、主题切换、Logo 持久化等增强能力，目标是提高复用率与留存，但不阻塞首用价值。  
P2 或未来项包括云同步、社区、社媒直发、AI 配色、地理可视化等，这些方向潜在价值高，但成本和不确定性显著更高，且会拉长当前迭代周期。  
取舍逻辑是：先确保“能稳定交付”，再做“更好看、更智能、更联网”。

## 未做的需求（为什么不做）
1. **账号体系与云同步**：当前定位是本地优先工具，先保证隐私和稳定性；云化会带来开发、运维与合规成本。  
2. **一键社媒发布**：跨平台接口和审核策略复杂，短期 ROI 不清晰，且偏离“本地封装”主目标。  
3. **AI 自动配色与智能推荐**：在路线图中，但目前缺少可验证的用户收益数据与质量基线。  
4. **更广格式与深度色彩管理能力**：如 TIFF/HEIC/ICC 深度处理仍在规划，属于专业能力升级而非首发必需。  
5. **完整商业化闭环（付费墙与支付）**：已有策略草案，但真实定价、转化与渠道数据仍需验证【待补充】。

# 用户主流程（5步以内）
1. **导入**：用户拖拽单张或多张照片进入应用，底部胶卷带立即出现可选图列表。  
2. **自动识别**：系统自动读取 Exif（品牌、机型、镜头、曝光参数）并尝试匹配品牌 Logo。  
3. **快速定型**：用户在 Classic/Cinema/Float 三种模板中选一套，必要时调比例、边框、字体、颜色。  
4. **一致性处理**：用户可将当前配置“应用到所有”，确保批量图片视觉风格统一。  
5. **导出交付**：用户单图保存或批量导出，完成可发布成品封装。

## 关键页面/关键交互说明（用文字描述，不依赖图片）
界面结构遵循“控制-预览-信息-列表”四区分工。左侧是样式与模板控制，承担“决策输入”；中间是画布实时预览，承担“结果确认”；右侧是 Exif 与导出设置，承担“参数校验与交付动作”；底部是胶卷带，承担“批量切换和上下文保持”。  
关键交互有三点：第一，拖拽即开始，降低入口门槛；第二，实时预览 + 轻量微调，避免“导出后才知道不对”；第三，“应用到所有 + 批量导出”，把批处理从重复手工改成一次性配置。Logo 交互中同时提供自动识别、循环切换和上传替换，确保“默认智能”与“人工可控”并存。

## 体验原则（3–5条）
1. **先给结果，再给控制**：先自动识别与默认模板，用户在可见结果上再做小调整。  
2. **减少重复操作**：能自动推断的就不让用户重复输入，能批量应用的就不让用户逐张设置。  
3. **一致性优先于自由度**：先保证可交付质量稳定，再逐步开放更复杂的自由编辑能力。  
4. **本地安全优先**：默认本地处理，不把“隐私风险”转嫁给用户。  
5. **异常可恢复**：任何自动化失败都给手动兜底，不让流程中断。

## 异常与兜底（失败/无结果/成本超限/不可用时怎么处理）
1. **Exif 读取不全或缺失**：允许用户继续导出，并保留可编辑文本位，避免因数据缺失阻断流程。  
2. **品牌 Logo 未匹配到**：自动回退为文本模式，用户可手动上传 Logo 或继续用文字展示。  
3. **批量导入中个别文件失败**：跳过异常文件并继续后续任务，避免“一个失败拖垮全部”。  
4. **导出路径不可写或权限不足**：通过系统目录选择与错误提示引导用户重选路径。  
5. **大图处理带来的性能压力**：采用单活跃大图与缓存策略控制内存开销，优先保证编辑过程可响应。  
6. **模板配置冲突或导入异常**：模板导入失败时给出格式错误提示，避免污染已有配置。

# 指标体系（北极星 → 漏斗）
建议以“交付成功效率”作为北极星：用户是否能在短时间内完成高质量导出。该目标在现有文档里可拆成三类指标：  
1. **激活指标**：TTFV（首次价值时间）与首次导出成功率。  
2. **留存指标**：周使用频次、D1/D7 回访率、二次打开率。  
3. **商业指标**：付费点击率、付费转化率、功能解锁使用率。  
漏斗建议统一为：曝光 → 访问/下载 → 安装启动 → 首次导出 → 周内复用 → 付费转化。当前仓库内缺少统一的真实数值口径，除版本记录外，多数指标数据为【待补充】。

## 风险与约束（合规/成本/体验/增长）

4. **体验风险**：高分图批处理存在性能与稳定性压力，需持续进行内存与渲染优化。  
5. **隐私与合规风险**：元数据可能包含位置信息，需明确默认策略与用户可见开关。  
6. **经营风险**：当前缺少完整的量化看板，策略优先级判断仍有不确定性。

# FAQ（至少 15 问）

## 这是什么/适合谁
### Q1. RZFrame 是什么？（是什么/做什么/解决什么问题）
A1. RZFrame 是一个桌面端 EXIF 边框与作品封装工具，核心解决“修图后到发布前”这一步效率低、风格不一致的问题。

### Q2. 这个工具适合谁？（适用人群/谁最该用）
A2. 主要适合职业摄影师、摄影博主、胶片爱好者和器材评测者，尤其是有批量交付或稳定风格需求的人。

### Q3. 哪些人可能不适合？（不适合谁/不建议谁用）
